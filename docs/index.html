The claim

Run a coding agent — or ten of them, or a hundred — on the same repo without your laptop melting. Each agent gets its own real workspace, materialized cheaply via APFS clonefile(2) (macOS) or cp --reflink (Linux btrfs / xfs / zfs).

How it works

  1. Skip cloning .git/. Use git worktree add --no-checkout to set up the worktree admin file, exactly like the standard tool. The source repo's object store is shared at zero cost.
  2. Clone the working tree via the kernel. Each top-level entry under the source goes through one clonefile(2) call (recursive on directories). On macOS APFS this is metadata-only; the underlying blocks remain shared until something writes to them.
  3. Populate the index with git read-tree HEAD. No file I/O; just makes git status see a clean tree. ~30 ms.

Three steps, ~160 ms per workspace at warm cache on a 10k-file repo. The benchmark numbers above include all three.

For AI agents

Every command supports --json for machine output and --agent for an agent-friendly summary with a next_command field. Drop AGENTS.md and skills/ivk/ into your repo (or let ivk init --agent-instructions do it) and an agent that's never heard of ivk can follow the workflow without prior knowledge.

$ ivk doctor --agent --json
{
  "ok": true,
  "command": "doctor",
  "next_command": "ivk new <task-name>",
  "recommended_next_steps": [...],
  "inside_ivk_workspace": false,
  "repo_initialized": true,
  ...
}

See the full recorded session transcript — every step an agent takes to fix a tiny off-by-one bug, with the literal JSON output captured.
